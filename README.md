# afischh

Building tools, language work, and security research at the boundary of
agents and AI systems.

## Focus

- **MAD Analysis** — Model-Agent-Destruction. A discipline I'm shaping,
  treating vulnerabilities in AI agents and LLM systems as clinical
  pathology rather than a sport. Living material forming at
  [brokenagent.site](https://brokenagent.site) (in preparation).

- **MCP ecosystem** — contributing to
  [modelcontextprotocol/python-sdk](https://github.com/modelcontextprotocol/python-sdk)
  with auth-layer fixes; building MCP-native research tools for agent
  observability and attack-surface mapping.

- **Language work** — Nim, Common Lisp, occasional contributions to
  compilers and stdlibs of small-but-serious languages. Long-term work
  on a Lisp+Unix coprocessor-language (NOL), currently expressed as a
  book rather than a runtime.

- **Open source** — selective contributions to
  [Nim](https://github.com/nim-lang/Nim) (compiler/stdlib),
  [Monocle](https://github.com/monocle2ai/monocle) (GenAI tracing
  framework by Okahu, Linux Foundation Sandbox project),
  [MCP Python SDK](https://github.com/modelcontextprotocol/python-sdk),
  and others.

## Active projects

- **ai_nima** — persistent Nim runtime daemon designed to be subordinate
  to a Claude Code session
  ([repo](https://github.com/afischh/ai_nima)).
- **MAD Tractatus** — propositional book of AI agent security patterns,
  in preparation, will publish at brokenagent.site.
- **MCP security toolkit** — small focused tools for static and
  black-box analysis of MCP servers, in preparation.

## How I work

I pair-code with Claude Code (Anthropic) as a primary engineering
partner. Every PR carries explicit disclosure of this. Slower on writing,
faster on architecture and review.
